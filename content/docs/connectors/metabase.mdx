## Before you begin

### Source

1. Confirm the Metabase instance is reachable from the tapstate environment, whether it is self-hosted or cloud-registered.
2. Create a least-privilege Metabase account for the intended analytics content.
3. Record the full HTTP or HTTPS base URL, including a non-default port when applicable.

Grant the account access only to the Metabase collections and content the pipeline needs.

## Create a connection

```yaml title="tapstate-work/source/finance_metabase.tapstate.yml"
version: tapstate/v1
kind: source
id: finance_metabase
connector: metabase
config:
  username: ${METABASE_USERNAME}
  password: ${METABASE_PASSWORD}
  matebaseDomainName: https://metabase.internal.example
mode: api
```

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- This connector reads Metabase content only; it does not write records or apply schema changes.
- Use an HTTPS base URL when Metabase requires encrypted transport.
- Metabase permissions and API limits are evaluated by the service at runtime.

## Reference

| Field | Required | Use |
|---|---|---|
| `username` | Yes | Metabase login account. |
| `password` | Yes | Login password; inject through an environment variable. |
| `matebaseDomainName` | Yes | Full Metabase HTTP or HTTPS base URL. Use this exact field spelling. |
