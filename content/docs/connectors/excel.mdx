## Before you begin

### Source

<SourceModeTabs label="Choose the Excel source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to read the matching workbooks once. Place them where the tapstate environment can reach them, then test the intended sheet structure with a representative XLS or XLSX workbook, including any merged cells or formulas.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** when new or changed workbooks should be picked up after the initial read. The connector scans the configured path every 60 seconds. New and modified workbooks are re-read; deleting a workbook does not emit row deletions.

</SourceModeTab>
</SourceModeTabs>

## Create a connection

```yaml title="tapstate-work/source/monthly_excel.tapstate.yml"
version: tapstate/v1
kind: source
id: monthly_excel
connector: excel
config:
  protocol: local
  encoding: UTF-8
  filePathString: /data/import/monthly-report.xlsx
mode: snapshot
```

For SFTP, SMB, S3FS, or OSS, add only the matching transport fields described in Reference.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- This connector reads Excel files. Target writes and schema changes are not included.
- XLS and XLSX are supported, including workbooks with merged cells and formulas. Verify formula results and sheet layout with a representative workbook.
- CDC detects new and modified workbooks on a 60-second scan. Workbook or row deletion is not propagated.
- Keep credentials outside committed workspace files and test memory use with production-sized workbooks.

## Reference

| Protocol | Fields |
|---|---|
| Common | `protocol`, `encoding`, `filePathString` |
| FTP | `ftpHost`, `ftpPort`, `ftpUsername`, `ftpPassword`, `ftpPassiveMode`, `ftpConnectTimeout`, `ftpDataTimeout` |
| SFTP | `sftpHost`, `sftpPort`, `sftpUsername`, `sftpPassword` |
| SMB | `smbHost`, `smbUsername`, `smbPassword`, `smbShareDir` |
| S3FS / OSS | `accessKey`, `secretKey`, `endpoint`, `region`, `bucket` |

`protocol` accepts `local`, `ftp`, `sftp`, `smb`, `s3fs`, and `oss`; `encoding` defaults to `UTF-8`.

### Data compatibility

Excel schema inference supports `BOOLEAN`, `DATE`, `DOUBLE`, `STRING`, and `TEXT`.
