## Before you begin

### Source

Create a least-privilege RocketMQ account with read access to the intended topics. Confirm the NameServer endpoint, port, and any TLS policy with the RocketMQ administrator.

### Target

Grant a dedicated writer permission only for the destination topics. Pre-create them with the queue and retention policy expected by downstream consumers.

## Create a connection

```yaml title="tapstate-work/source/events_rocketmq.tapstate.yml"
version: tapstate/v1
kind: source
id: events_rocketmq
connector: rocketmq
config:
  mqTopicString: orders.events
  mqHost: nameserver.rocketmq.internal
  mqPort: "9876"
  mqUsername: ${ROCKETMQ_USER}
  mqPassword: ${ROCKETMQ_PASSWORD}
  useTLS: false
mode: stream
```

For a target-only connection, omit `mode`. Set `useTLS: true` only when the endpoint and runtime are configured for it.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- Test message ordering, retries, consumer groups, and delivery guarantees before production. Schema changes are not applicable to this stream connector.
- `useTLS` enables encrypted transport. Test certificate trust and hostname verification with the broker endpoint before production.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `mqTopicString` | No | — | Optional topic selection. |
| `mqHost`, `mqPort` | Yes | — | RocketMQ NameServer host and port. |
| `mqUsername`, `mqPassword` | No | — | Account credentials; use an environment variable for the password. |
| `useTLS` | No | `false` | Boolean TLS setting. |
