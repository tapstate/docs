## Before you begin

### Target

Create a dedicated Databend account with rights to write the destination database. Grant table-creation privileges only when automatic table management is explicitly approved.

## Create a connection

```yaml title="tapstate-work/source/warehouse_databend.tapstate.yml"
version: tapstate/v1
kind: source
id: warehouse_databend
connector: databend
config:
  host: databend.internal
  port: 8000
  database: analytics
  user: ${DATABEND_USER}
  password: ${DATABEND_PASSWORD}
```

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- This connector writes to Databend only. Source reads, CDC, and schema-change handling are not included.
- The connector is Databend 0.9+. Verify exact server version, target types, and privileges before production use.

## Reference

| Field | Required | Use |
|---|---|---|
| `host` | Yes | Databend host. |
| `port` | Yes | Databend port; field type is number. |
| `database` | Yes | Destination database. |
| `user` | Yes | Database user. |
| `password` | No | Password; use an environment variable. |
